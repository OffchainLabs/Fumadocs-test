Arbitrum Nitro is the software that powers all Arbitrum chains. This how-to shows how you can build a Docker image, or binaries, directly from Nitro's source code.

This how-to assumes that you're running one of the following operating systems:

- [Debian 11 (bullseye)](https://www.debian.org/releases/bullseye/)
- [Ubuntu 22.04 (amd64)](https://releases.ubuntu.com/jammy/)
- [MacOS Ventura 13](https://developer.apple.com/documentation/macos-release-notes/macos-13-release-notes).

## Build a Docker image

### Step 1. Configure [Docker](https://docs.docker.com/engine/install)

Install Docker for your platform following the official documentation.

### Step 2. Download the Nitro source code

```shell
git clone --branch v2.3.4 https://github.com/OffchainLabs/nitro.git
cd nitro
git submodule update --init --recursive --force
```

### Step 3. Build the Nitro node Docker image

```shell
docker build . --tag nitro-node
```

That command will build a Docker image called `nitro-node` from the local source.

## Build Nitro's binaries natively

If you want to build the node binaries natively, execute Steps 1-3 of the [Build a Docker image](#build-a-docker-image) section and continue with the steps described here.

### Step 4. Configure node [18](https://github.com/nvm-sh/nvm)

```shell
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.3/install.sh | bash
source "$HOME/.bashrc"
nvm install 18
nvm use 18
```

### Step 5. Configure [Rust](https://www.rust-lang.org/tools/install)

```shell
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source "$HOME/.cargo/env"
rustup install 1.75.0
rustup default 1.75.0
rustup target add wasm32-unknown-unknown --toolchain 1.75.0
rustup target add wasm32-wasi --toolchain 1.75.0
cargo install cbindgen
```

### Step 6. Configure Go [1.20](https://github.com/moovweb/gvm)

```shell
bash < <(curl -s -S -L https://raw.githubusercontent.com/moovweb/gvm/master/binscripts/gvm-installer)
source "$HOME/.gvm/scripts/gvm"
gvm install go1.20
gvm use go1.20 --default
```

### Step 7. Start build

```shell
make
```

### Step 8. Run your node

To run your node using the generated binaries, use the following command from the `nitro` folder, with your desired parameters

```shell
./target/bin/nitro <node parameters>
```
